# Управление очередью автомойки (MVP)

Современная система живой очереди автомойки в реальном времени. Включает кабинет владельца автомойки, панель оператора (планшет администратора) и клиентскую мобильную панель отслеживания статусов.

## Стек технологий
- **Фронтенд**: React (Vite)
- **Стилизация**: Custom Vanilla CSS (премиальный темный стек с поддержкой Glassmorphism и адаптивной мобильной версткой)
- **База данных и Realtime**: Supabase (подключение по JS SDK, репликация реального времени)
- **Хостинг**: Vercel

---

## 🛠️ Настройка Supabase (База данных)

1. Создайте проект на [Supabase](https://supabase.com/).
2. Перейдите во вкладку **SQL Editor** и выполните SQL-команды из файла [supabase_schema.sql](file:///supabase_schema.sql) в корне проекта. Это создаст таблицы `businesses` и `queue`, а также настроит политики публичного доступа (RLS) для MVP.
3. **ВАЖНО (Realtime)**: Чтобы очередь и статусы обновлялись мгновенно без перезагрузки страниц, включите репликацию:
   - В панели Supabase перейдите в **Database** -> **Replication**.
   - Нажмите на публикацию `supabase_realtime` (или измените существующую).
   - Включите (переведите в состояние ON) таблицы `businesses` и `queue`.

## 🚀 Локальный запуск проекта

Для запуска проекта локально выполните следующие шаги:

1. Установите зависимости:
   ```bash
   npm install
   ```
2. Запустите локальный сервер разработки:
   ```bash
   npm run dev
   ```
3. Откройте в браузере предоставленную ссылку (например, `http://localhost:5173`).
4. При первом запуске система отобразит **страницу конфигурации Supabase**. Скопируйте туда:
   - **Project URL** (из раздела Settings -> API в Supabase)
   - **Anon Public API Key** (оттуда же)
   Эти данные сохранятся в вашем браузере (`localStorage`). Вы также можете создать файл `.env` в корневом каталоге для постоянного хранения ключей в коде (см. ниже).

### Настройка через `.env` (опционально)
Создайте файл `.env.local` в корне проекта и укажите ваши ключи:
```env
VITE_SUPABASE_URL=https://your-project-id.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-public-key
```

## 🌎 Развертывание на Vercel

Этот проект готов к переносу в GitHub и мгновенному импорту в Vercel:

1. Создайте репозиторий на GitHub и загрузите туда файлы проекта:
   ```bash
   git init
   - Добавьте файлы в коммит и запушьте на GitHub
   ```
2. Авторизуйтесь на [Vercel](https://vercel.com/) и импортируйте созданный репозиторий.
3. В разделе **Environment Variables** (переменные окружения) при создании проекта добавьте две переменные:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
   Укажите соответствующие значения из консоли Supabase.
4. Нажмите **Deploy**. Готово!

## 📱 Роли и сценарии работы

Система работает на одной кодовой базе, разделяя экраны с помощью параметров запроса (Query Params):

### 1. Кабинет владельца
- **Ссылка**: `?role=owner`
- **Описание**: Позволяет настроить количество боксов, адрес и название мойки, а также базовое время мытья.
- **QR-код**: Во вкладке "QR-код и ссылки" генерируется динамический QR-код. 
  - Кнопка **Скачать PNG** сохраняет изображение.
  - Кнопка **Печать А4 (PDF)** вызывает диалоговое окно печати браузера с готовой сверстанной A4-листовкой (все лишние элементы панели скрываются, оставляя только красивый плакат с QR-кодом для клиентов).
- **Аналитика**: Выводит общее количество обслуженных машин за сегодня/неделю, среднее время мойки одной машины (рассчитанное по кликам оператора) и количество отказов («отвалов»).

### 2. Панель оператора (Администратора автомойки)
- **Ссылка**: `?role=operator&id=BUSINESS_ID` (Ссылка генерируется в кабинете владельца)
- **Описание**: Отображается на планшете или ноутбуке на мойке.
  - **Блок боксов**: Каждая плитка показывает статус бокса (Свободен/Занят) и позволяет временно отключать боксы (например, на ремонт).
  - **Таймер**: Если бокс занят, на плитке горит реальный живой таймер обслуживания машины.
  - **Завершение мойки**: Кнопка «Мойка завершена» имеет модальное окно подтверждения ("ДА" / "ОТМЕНА") для исключения случайных кликов.
  - **Очередь**: Внизу выводится список машин в очереди.
  - **Ручное добавление**: Кнопка «Добавить вручную» позволяет записать приехавшую машину (без интернета) прямо с панели оператора. Она автоматически ставится в конец очереди со статусом «Готов» (Я на месте).
- **Алгоритм автоназначения**: Панель оператора автоматически распределяет клиентов из очереди. Как только один из боксов освобождается, система находит первого клиента в очереди со статусом "Ready" (Я на месте) и мгновенно переводит его в статус "In Box" (Заезжает в бокс), назначая бокс.

### 3. Клиентская панель (Мобильный браузер)
- **Ссылка**: `?role=client&id=BUSINESS_ID` (Сканируется через QR-код)
- **Описание**: 
  - **Регистрация**: Клиент видит сколько машин перед ним и примерное время ожидания. Вводит гос. номер и телефон и нажимает «Встать в очередь».
  - **Отслеживание**: Отображает номер в очереди, динамическое время ожидания, статус выполнения в виде прогресс-бара и кнопку «Покинуть очередь».
  - **Алерт 1 (Подтверждение присутствия)**: Когда перед клиентом остается 1-2 машины, на экране всплывает модальное окно с таймером обратного отсчета (60 секунд) и предложением нажать «Я на месте» или «Отменить». Если время истекло или нажата отмена, клиент удаляется из очереди (засчитывается отвал).
  - **Алерт 2 (Заезжайте)**: Когда бокс свободен и алгоритм назначает клиента, его мобильный экран перекрывается зеленым окном: «Ваша очередь! Заезжайте в Бокс №...».
